Sprint 26A — Stabilizzazione completa Alpha Chat (STOP nuove funzionalità)

Obiettivo: fermare temporaneamente lo sviluppo di nuove funzionalità e rendere Alpha Chat completamente stabile. Nessuna nuova feature. Solo bug fixing, audit, test e hardening.

Bug 1 — Messaggi “[Messaggio non decifrabile]” (CRITICO)

Analizzare l’intera pipeline Signal:

* Identity Key
* Signed PreKey
* One Time PreKeys
* X3DH
* Double Ratchet
* SessionBuilder
* SessionCipher
* SignalProtocolStore
* IndexedDB
* Cache decrypt
* Restore dopo refresh
* Restore dopo cambio pagina
* Restore dopo riapertura browser

Verificare che:

* le sessioni non vengano rigenerate inutilmente;
* le Identity Key restino persistenti;
* il decrypt parta solo quando Signal è completamente inizializzato;
* non esistano race condition tra login, caricamento chiavi e decrypt.

Aggiungere log diagnostici temporanei per individuare il punto esatto della rottura e rimuoverli al termine.

⸻

Bug 2 — Vocali muti / “Errore avvio” (CRITICO)

Analizzare:

* MediaRecorder
* MIME type Safari iOS
* AES-256-GCM
* Upload
* Download
* Decrypt
* Blob
* ObjectURL
* Audio element

Verificare che:

* il blob registrato contenga realmente audio;
* il blob cifrato e quello decifrato abbiano la stessa dimensione;
* Safari utilizzi un MIME realmente supportato;
* la riproduzione funzioni sempre.

Eliminare definitivamente “Errore avvio”.

Bug 5 — Chiamate non squillano (CRITICO)

Audit completo della pipeline chiamate.

Verificare ogni passaggio:

1. chiamante invia call.offer;
2. server WebSocket riceve;
3. server individua destinatario;
4. server invia call.offer;
5. client destinatario riceve;
6. CallContext aggiorna lo stato;
7. IncomingCallModal viene renderizzato;
8. startRing() viene eseguito;
9. lo squillo è realmente udibile.

Controllare inoltre:

* unlockNotifAudio()
* AudioContext
* HTMLAudioElement
* Safari iOS
* Chrome Android
* Desktop
* deviceId
* WsManager
* DND
* Busy mode
* Multi-device

Individuare il primo punto in cui la pipeline si interrompe.

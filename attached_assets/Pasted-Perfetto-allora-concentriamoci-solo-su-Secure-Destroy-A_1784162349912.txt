SPRINT 11 — SECURE DESTROY
OBIETTIVO
Alpha Chat introduce una funzione esclusiva chiamata:
🛡 Secure Destroy
Non è una semplice eliminazione del messaggio.
È una distruzione definitiva e irreversibile.
L'utente deve percepire che sta impartendo un ordine di distruzione.

IMPLEMENTAZIONE BACKEND
Secure Destroy deve eseguire realmente:
✓ eliminazione del documento MongoDB
✓ eliminazione di eventuali allegati
✓ eliminazione thumbnail
✓ eliminazione preview
✓ eliminazione storage Cloudflare R2
✓ invalidazione cache
✓ aggiornamento lista conversazioni
✓ aggiornamento ultimo messaggio
✓ aggiornamento unread counter
✓ broadcast WebSocket
evento
message.destroyed
====================================================================
DATABASE
Il messaggio NON deve rimanere nel database.
Non usare placeholder.
Non usare
"Messaggio eliminato"
Il record deve essere eliminato.
====================================================================
FRONTEND
Quando arriva
message.destroyed
il messaggio deve:
• dissolversi lentamente
• frammentarsi con un leggero effetto digitale
• scomparire completamente
L'animazione deve essere elegante.
====================================================================

mostrare

senza possibilità di recuperarlo.

OBIETTIVO
Quando un utente utilizza questa funzione non deve avere la sensazione di "eliminare un messaggio".
Deve avere la sensazione di impartire un ordine di distruzione definitiva.
Secure Destroy deve diventare una delle caratteristiche distintive del marchio Alpha Chat.

Una piccola idea di branding che trovo elegante: quando l’operazione termina con successo, invece di un semplice toast “Messaggio eliminato”, mostra per un secondo:

🛡 Secure Destroy completato

È un dettaglio, ma aiuta a rendere la funzione riconoscibile e coerente con l’identità di Alpha Chat.
